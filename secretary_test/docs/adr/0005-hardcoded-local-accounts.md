# ADR-0005: 하드코딩된 로컬 테스트 계정 (인증 서버 없음)

## 컨텍스트

앱은 다중 계정 간 데이터 격리와 계정 전환 기능이 필요하다. 실제 인증 서버를 구축하면 개발 복잡도와 운영 비용이 발생한다.

## 결정

인증 서버 없이 `LoginScreen.js`에 계정 6개를 하드코딩한다. 로그인 성공 시 `current_user_v1` 키에 사용자 객체를 저장하고 앱 시작 시 복원한다.

```
test@secretary.app / test1234    (테스트 계정)
admin@secretary.app / admin1234  (관리자)
kmj@secretary.app / test1234     (김민준, 삼성물산)
lsy@secretary.app / test1234     (이서연, 현대건설)
pjh@secretary.app / test1234     (박지훈, LG전자)
csa@secretary.app / test1234     (최수아, SK텔레콤)
```

## 이유

- 이 앱은 개인 사용·데모 목적이다. 실 사용자 계정 관리가 필요 없다.
- 다중 계정 데이터 격리(ADR-0004)를 테스트하려면 고정된 userId가 필요하다.
- 각 계정이 서로를 거래처로 인식하는 시나리오 테스트가 목적이다.

## 결과

- 긍정: 인증 서버 불필요, 오프라인 완전 동작, 계정 전환으로 다중 관점 테스트 가능
- 부정: 비밀번호 변경 불가, 실 배포 시 인증 계층을 전면 교체해야 함
- 제약: 각 계정은 자신의 이름을 거래처 목록에서 자동 필터링함 (본인이 본인의 거래처로 표시되는 것 방지)
