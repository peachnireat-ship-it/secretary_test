## 목적

스토리의 장면을 독자가 자연스럽게 읽어 내려가는 패널 시퀀스로 번역한다. 캐릭터 배치와 말풍선 공간을 명확히 정의하여 창작팀이 각자의 영역에서 충돌 없이 작업할 수 있도록 한다.

## 핵심 원칙

### 모바일 세로 스크롤 기준

웹툰은 **위에서 아래로** 스크롤한다. 모든 설계는 이 전제에서 출발:
- 패널 폭: 항상 화면 너비 100% (모바일 기준)
- 좌우 분할: 최대 2분할 (3분할 이상 = 가독성 저하)
- 감정 고조: 풀 와이드 패널 1개로 집중 (분산 금지)

### 감정 강도 = 패널 크기

| 감정 강도 | 패널 크기 |
|---------|---------|
| 일반 대화/이동 | 표준 (세로 30~40%) |
| 감정 발현/갈등 | 중간 확장 (세로 50~60%) |
| 감정 고조/클라이맥스 | 풀 와이드 (세로 70~100%) |
| 충격/반전 공개 | 단독 대형 패널 (세로 100%) |
| 코믹 반응 | 소형 분할 (세로 20~25%) |

### 시선 흐름 설계

기본 흐름: Z패턴 (좌→우, 아래→좌→우)

시선 유도 기법:
- **대각선 배치**: 캐릭터를 패널 대각선 위에 배치 → 자연스러운 시선 유도
- **말풍선 방향**: 말풍선 꼬리가 다음 패널 방향을 가리키면 독자 시선을 당긴다
- **클로즈업 → 풀샷**: 강조 → 맥락 순서. 반대 순서(풀샷 → 클로즈업)는 정보 → 감정 강조.
- **침묵 패널**: 말없는 패널은 독자가 멈추게 한다. 전략적으로 사용.

### 말풍선 공간 사전 계획

**대사 편집자가 후처리하는 게 아니라, 레이아웃 단계에서 미리 공간을 확보한다.** 공간 없는 곳에 대사를 억지로 넣으면 캐릭터 얼굴을 가린다.

대사 분량별 레이아웃 처리:
- **대사 없음**: 캐릭터 중앙, 배경 풀
- **대사 보통**: 상단 20~30% 공간 확보 (캐릭터 하단 배치) 또는 좌/우 여백
- **대사 많음**: 배경 단순화 + 캐릭터 하단 40% → 상단 60% 말풍선 공간
- **내레이션**: 최상단 또는 최하단 띠 형태로 분리

### 레이아웃 패턴 라이브러리

```
# 기본 패턴
[전체 와이드]                          ← 개막, 장면 전환, 클라이맥스
[좌 50%][우 50%]                      ← 대화, 대치
[좌 60%][우 40%]                      ← 주인공 강조 + 반응
[상단 와이드][하단 좌][하단 우]         ← 상황 설명 + 반응 2인
[소형 3연속]                           ← 빠른 템포, 연속 반응
```

## 출력 형식

**`_webtoon/{slug}/03_panels.md`:**

```markdown
# 패널 레이아웃
에피소드: {번호}  총 패널 수: {N}컷

## 장면 N — {제목}

### 패널 N-M
- 크기: 가로 100% / 세로 {비율}
- 구도: {설명}
- 카메라 앵글: {정면/측면/부감/앙각}
- 등장 인물 배치: {캐릭터명 — 위치, 원근}
- 말풍선 공간: {위치, 비율} → 대사 편집자에게 전달
- 효과선: {있음/없음, 방향}
- 시선 흐름: →{다음 패널 N-(M+1)}

## 전체 레이아웃 맵
```
[패널1: 풀 와이드]
[패널2: 좌 60%][패널3: 우 40%]
[패널4: 풀 와이드]
