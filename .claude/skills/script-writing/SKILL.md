## 대본 구조

### 훅 (0~30초) — 가장 중요

시청자가 떠나는 80%가 첫 30초 안에 일어난다. 훅은 계속 볼 이유를 즉시 제시한다.

**효과적인 훅 패턴:**
- **충격적 사실**: "대부분의 사람들이 모르는 사실이 있습니다. {핵심 포인트}."
- **문제 제기**: "혹시 {시청자 공통 문제}를 경험하고 계신가요?"
- **결과 약속**: "이 영상을 끝까지 보시면 {구체적 혜택}을 얻으실 수 있습니다."
- **카운트다운**: "오늘 소개할 {숫자}가지 방법 중 마지막 것이 가장 강력합니다."

훅에서 클릭베이트는 피한다 — 훅과 다른 본론은 시청 이탈과 알고리즘 페널티를 유발한다.

### 본론

**정보 밀도 원칙**: 불필요한 서론("오늘 채널에 오신 것을 환영합니다")은 제거한다. 바로 약속한 내용을 전달한다.

**구조 템플릿 (정보형 영상):**
```
1. 핵심 주장/결론 먼저 (두괄식)
2. 근거 + 예시
3. 실천 방법
4. 요약 + 다음 포인트 예고
```

**시청 유지 기법:**
- 단락 전환마다 "그런데", "여기서 중요한 건", "하지만 반전이 있습니다" 등 전환 신호 사용
- 3~5분마다 "지금까지 X를 알아봤고, 다음은 Y입니다"로 진도 확인
- 중간 CTA: "이게 도움됐다면 좋아요 눌러주세요"를 중반부(50~60%) 지점에 1회 삽입

**분량 기준 (영상 길이별):**
| 영상 길이 | 대본 단어 수 |
|----------|------------|
| 5분 | 약 800단어 |
| 10분 | 약 1500단어 |
| 15분 | 약 2500단어 |

### CTA (마지막 30초)

하나만 선택해서 요청한다 (세 가지 동시 요청은 효과 없음):
- 구독 요청 — 채널 성장 목표일 때
- 댓글 요청 — 커뮤니티 참여 목표일 때 (댓글 주제를 구체적으로 제시)
- 다음 영상 유도 — 시리즈물일 때 ("다음 편에서 X를 다룹니다")

## 대본 형식

```
[제목/주제]
예상 영상 길이: X분
---

[훅]
(대본 텍스트)
[B-roll: 시각자료 설명]

[본론]
## 섹션 1: {소제목}
(대본 텍스트)
[화면: 예시 이미지/동영상]

## 섹션 2: {소제목}
(대본 텍스트)
[자막: 강조할 수치나 단어]

[중간 CTA] ← 전체 분량 50~60% 지점
(대본 텍스트)

[CTA]
(대본 텍스트)
```

## 톤 가이드

- **교육형**: 명확하고 단계적, 전문 용어는 즉시 설명
- **엔터테인먼트형**: 개인 스토리, 유머, 과장 허용
- **리뷰형**: 장단점을 균형 있게, 주관적 의견임을 명시
- 어느 톤이든 구어체로 작성한다 — 읽히는 문장이 아니라 말하는 문장
