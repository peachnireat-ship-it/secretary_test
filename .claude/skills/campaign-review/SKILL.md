## 검증 철학

캠페인 QA의 핵심은 각 산출물의 내용 검토가 아니라 **단계 간 경계면에서 발생하는 충돌 감지**다.

- 시장 조사가 "25~34세 직장인"이라 했는데 카피가 "퇴직 후 여유로운 삶"을 소구하면 → 경계면 오류
- 카피가 "따뜻하고 감성적"인데 비주얼이 "냉철한 다크톤"이면 → 경계면 오류
- 비주얼 방향이 2개인데 A/B 테스트가 3버전을 계획하면 → 경계면 불일치

이 충돌들은 각 파일을 개별로 읽으면 발견하기 어렵다. 반드시 두 파일을 동시에 읽으며 쌍으로 비교한다.

## 경계면 교차 검증 체크리스트

### 경계면 1: 페르소나 ↔ 카피 톤

| 검증 항목 | 페르소나 속성 | 카피에서 확인 |
|---------|------------|------------|
| 연령대 반영 | 20대 → 구어체/이모지 허용 | 카피 어조가 연령대와 일치? |
| 페인포인트 소구 | 페르소나의 주요 불만 | 카피가 해당 페인포인트를 다루는가? |
| 채널 선호 | 인스타/유튜브/카카오 | 채널별 카피가 채널 문화에 맞는가? |

→ 불일치 발견 시 **🔴 Critical** (페르소나-카피 불일치는 캠페인 전체 방향 오류)

### 경계면 2: 카피 감성 ↔ 비주얼 분위기

카피의 핵심 감성 키워드(02에서 추출) vs. 비주얼의 색상·이미지 방향(03에서 추출):

| 카피 감성 | 비주얼 기대치 | 충돌 예시 |
|---------|-----------|--------|
| 따뜻함·친근함 | 웜톤, 라이프스타일 | 냉철한 다크톤 → 충돌 |
| 프리미엄·신뢰 | 미니멀, 무채색+골드 | 붐비는 색상, 캐릭터 → 충돌 |
| 유머·경쾌함 | 밝은 채도, 일러스트 | 무거운 세리프, 다크 → 충돌 |

→ 충돌 발견 시 **🔴 Critical**

### 경계면 3: 비주얼 변형 수 ↔ A/B 테스트 변수

비주얼 방향 수(03 파일): N개
A/B 계획의 비주얼 관련 변수(04 파일): M개

N ≠ M이면 **🟡 Major** (재료 없는 테스트 계획)

### 경계면 4: 채널 목록 일관성

01~04 파일이 모두 동일한 채널을 다루는지 확인:
- 01에 인스타 타깃 언급 → 02에 인스타 카피 있어야 함 → 03에 1:1 포맷 있어야 함 → 04에 인스타 테스트 포함
- 채널 누락 발견 시 **🟡 Major**

## 개별 섹션 품질 기준

| 항목 | 기준 | 심각도 |
|------|------|-------|
| 헤드라인 변형 수 | 최소 3개 | 🟡 Major |
| CTA 행동 동사 포함 | "시작", "받기", "체험" 등 | 🟡 Major |
| 비주얼 hex 코드 | 색상마다 #코드 명시 | 🟢 Minor |
| A/B 가설 측정 가능성 | "N% 향상" 형태 | 🟡 Major |
| 샘플 크기 근거 | 신뢰도/검출력 기준 명시 | 🟢 Minor |
| 경쟁사 분석 최소 2개 | 시장 조사에 | 🟢 Minor |

## 점수 산정

| 항목 | 감점 |
|------|------|
| 🔴 Critical 1건 | -15점 |
| 🟡 Major 1건 | -5점 |
| 🟢 Minor 1건 | -2점 |
| 섹션 누락 1개 | -3점 |
| 기준점 | 100점 |

**60점 미만:** 재작업 권고. 핵심 개선 3항목 목록 필수.
**80점 이상:** 캠페인 패키지 완성도 충분.

## 최종 패키지(06_campaign_package.md) 구성 원칙

요약 테이블은 의사결정자가 30초 안에 캠페인 전체를 파악할 수 있도록 작성한다:
- 타깃, 핵심 메시지, 추천 헤드라인 1개, 추천 비주얼 방향 1개, 1순위 A/B 테스트만 요약
- 상세 내용은 01~05 파일 참조로 위임
- QA에서 발견된 수정 사항을 직접 반영하여 01~04 오류를 패키지에서 교정
