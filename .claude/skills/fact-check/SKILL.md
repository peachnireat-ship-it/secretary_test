## 검증 절차

### Step 1: 주장 추출

각 조사 파일에서 검증 가능한 사실 주장을 목록화한다:
- "A가 B를 발표했다" 형태의 사실 주장
- 수치·날짜·이름이 포함된 주장
- 인과 주장 ("~이기 때문에 ~하다")

커뮤니티 의견("~하면 좋겠다", "~인 것 같다")은 사실 주장과 분리한다.

### Step 2: 교차 비교

동일한 주장이 여러 소스에 나타나는지 확인한다:
- **일치**: 2개 이상 독립 소스가 동일 → 신뢰도 높음 ✅
- **부분 일치**: 일부 다름 → 신뢰도 중간 ⚠️, 다른 내용 명시
- **불일치**: 소스마다 다름 → 신뢰도 낮음 ❌, 모두 출처와 함께 병기

### Step 3: 추가 검증

불일치 또는 중요 주장은 WebSearch로 제3 출처 확인:
- 공식 발표문·법률·원시 통계 데이터 우선
- 확인 후 신뢰도 재평가

### Step 4: 편향 탐지

- 단일 소스만 언급하는 극단적 주장 → ⚠️ 주의 표시
- 광고성·이해관계자 주장 → 제외 또는 별도 표시
- 커뮤니티 반응 = 의견, 사실로 처리하지 않음

## 신뢰도 등급

| 등급 | 기준 |
|------|------|
| ✅ 높음 | 2개 이상 독립 소스 일치, 공식 출처 포함 |
| ⚠️ 중간 | 단일 소스이거나 소스 간 부분 불일치 |
| ❌ 낮음 | 소스 불일치 또는 출처 불명확 |

## 출력 형식

```markdown
# 교차 검증 결과: {주제}
검증일: {날짜}

## ✅ 검증된 사실 (신뢰도 높음)
- {주장} — 확인 소스: [{출처1}], [{출처2}]

## ⚠️ 주의 사항 (신뢰도 중간)
- {주장} — 확인 소스: [{출처}], 주의 이유: {이유}

## ❌ 불확실·불일치 (신뢰도 낮음)
- {주장} — 소스A: ..., 소스B: ..., 판단 불가 이유

## 제외된 주장
- 광고성·미확인·이해관계 충돌 항목 목록

## 검증 범위
- 검증한 사실 주장 수: ...
- 추가 검색한 항목 수: ...
- 누락된 조사 파일 (있다면): ...
```
